This API configures (CREATE/SET/DESTROY) an egress modifier template (EMT). An EMT is used to modify packets using ACL actions. An EMT needs to be allocated first (CREATE cmd) before its parameters are set (SET cmd). Note: an EMT used in an ACL action cannot be destroyed. Supported devices: Spectrum2, Spectrum3. Parameters:

Parameters
[in]handle- SX-API handle
[in]cmd- CREATE / SET / DESTROY
[in]emt_id- The egress modifier template to configure
[in]emt_cfg_p- Pointer to the configuration to apply to the EMT (ignored by CREATE & DESTROY commands).
Returns
SX_STATUS_SUCCESS if operation completes successfully.
SX_STATUS_PARAM_ERROR if any input parameter is invalid.
SX_STATUS_ENTRY_ALREADY_EXISTS if trying to create an EMT that already exists.
SX_STATUS_RESOURCE_IN_USE when trying to destroy an EMT used by an ACL action.
SX_STATUS_UNSUPPORTED if configuration is not supported by the platform*
SX_STATUS_ERROR general error.

This API configures the below egress modifier attributes according to the attr_type of the listed objects (att_cfg_list_p):

a. port attributes (attr_type=TYPE_PORT_E): payload_offset_shift- By default the flex packet modification is supported up to the byte where the actual packet header parsing stops (actual parsing offset). also the actual parsing offset is less or equal to the configure parsing depth. the offset configuration enables to extend the actual parsing offset in order to apply either payload modification (EDIT) or additional header information element to the packet (PUSH) by the flex egress modifier. E.g : [---—default valid for flex modification-—] [---—default actual parsing offset+------—]------------------------------------------------—+ |Ethernet [18B] | ipv4[20B] | udp[8B] | Payload | +--------------—+--------------—+------—+------------------------------------------------—+

configuring payload_offset_shift of 8 bytes:

[---------------—valid range for flex modification------------------—] [---—default actual parsing offset+------—]-------------------—Payload-------------------—+ |Ethernet [18B] | ipv4[20B] | udp[8B] | payload_offset_shift[8B] | | +--------------—+--------------—+------—+------------------------------------------------—+

Notes:

  • The configured offset is limited by up to 512B (in 2B granularity).
  • The default configuration of the payload_offset_shift is - 0 (the payload starts at the end of the actual parsing).
  • payload_offset_shift configuration is per port (physical ports only), split ports will not inherit the configuration from the original split port.
  • in case of LAG membership, all port members should be configured with the same payload_offset_shift.
  • Supported devices: Spectrum2, Spectrum3.

Parameters:

Parameters
[in]handle- SX-API handle.
[in]cmd- SX_ACCESS_CMD_ADD / SX_ACCESS_CMD_DELETE.
[in]*attr_cfg_list_p- pointer to a list of port/common attributes to set/unset
[in]attr_cfg_cnt- number of elements in the list.
Returns
SX_STATUS_SUCCESS - operation completes successfully.
SX_STATUS_PARAM_NULL - referenced inputs are NULL.
SX_STATUS_CMD_UNSUPPORTED - CMD is not supported.
SX_STATUS_CMD_ERROR - received struct is not the expected
SX_STATUS_PARAM_ERROR - input parameter is invalid.
SX_STATUS_PARAM_EXCEEDS_RANGE - input parameter is not in the expected range.
SX_STATUS_ENTRY_NOT_FOUND - data base call back if entry can't be retrieved.
SX_STATUS_NO_MEMORY - data base call back in case of memory allocation error.
SX_STATUS_NO_RESOURCES - memory allocation failure.
SX_STATUS_UNSUPPORTED - configuration is not supported by the platform.
SX_STATUS_MODULE_UNINITIALIZED- data base initialization failure.
SX_STATUS_ERROR - general error.
