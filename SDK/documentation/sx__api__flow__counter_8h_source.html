243 /*
244  * This API supports to CREATE/SET/DESTROY a profile with specific params.
245  * For CREATE:
246  * Create a profile with specific hash params and bin group size. If the profile key exists, API will fail and return
247  * "SX_STATUS_ENTRY_ALREADY_EXISTS".
248  * 1) "hash_params" contains various flow estimator profile hash params.
249  * - If "hash_params" is not NULL, those hash params will overwrite the profile's current params;
250  * - if it is NULL, default hash params (with only SIP/DIP/SPort/DPort/Protocol fields be enabled) will be set.
251  * 2) bin_group_size specifies the number of flow estimator bins (buckets) for flow cardinality calculations.
252  * - The bin_group_size should only be set with values of 16, 32, 64, 128, 256, 512, other values are
253  * invalid and would be refused with "SX_STATUS_PARAM_EXCEEDS_RANGE"
254  * - bin_group_size also determines counter_num of flow counter to be allocated (via sx_api_flow_counter_bulk_set)
255  * and bound with the profile in an ACL action, i.e.,
256  * counter_num = bin_group_size / FLOW_ESTIMATOR_BINS_PER_FLOW_COUNTER
257  * where FLOW_ESTIMATOR_BINS_PER_FLOW_COUNTER is a macro 4.
258  * Example:
259  * - Users call this API to create a profile with bin_group_size = 64.
260  * - Users should calculate the required counter_num, which should be 16 (=64/4),
261  * - Users call sx_api_flow_counter_bulk_set() to create the specific flow counters,
262  * with counter_num = 16, type = SX_FLOW_COUNTER_TYPE_ESTIMATOR.
263  *
264  * For SET:
265  * Set a profile with specific hash params and bin group size. If the profile key is not created, fails with
266  * "SX_STATUS_ENTRY_NOT_FOUND".
267  * 1) "hash_params" contains profile hash params.
268  * - If "hash_params" is not NULL, those hash params will overwrite the profile's current params;
269  * - If it is NULL, default hash params (with only SIP/DIP/SPort/DPort/Protocol fields be enabled) will be set.
270  * 2) bin_group_size specifies the number of flow estimator bins (buckets) for cardinality calculations.
271  * - If the profile is in use, its bin_group_size cannot be changed. In that case, the input value
272  * should be equivalent to the profile's existing value. Otherwise, it fails with "SX_STATUS_RESOURCE_IN_USE".
273  * - Other description please see bin_group_size above in "CREATE".
274  *
275  * For DESTROY:
276  * Un-allocate the specific profile key. If the profile key is not created, fails with "SX_STATUS_ENTRY_NOT_FOUND".
277  * If the profile key is still used in some ACL actions, fails with "SX_STATUS_RESOURCE_IN_USE".
278  *
279  * Supported devices: Spectrum2, Spectrum3, Spectrum4.
280  *
281  * @param[in] handle - SX-API handle
282  * @param[in] cmd - CREATE/SET/DESTROY
283  * @param[in] profile_key - flow estimator profile key
284  * @param[in] profile_cfg_p - flow estimator profile parameters. Ignored and can be NULL for cmd = DESTROY
285  *
286  *
287  * @return SX_STATUS_SUCCESS if operation completes successfully
288  * @return SX_STATUS_PARAM_NULL or SX_STATUS_PARAM_EXCEEDS_RANGE or SX_STATUS_PARAM_ERROR if any input parameter is invalid
289  * @return SX_STATUS_NO_MEMORY if no memory is available
290  * @return SX_STATUS_CMD_ERROR if invalid command is passed or internal RPC mechanism to SDK server fails
291  * @return SX_STATUS_INVALID_HANDLE if called Invalid Handle
292  * @return SX_STATUS_ERROR general error
293  * @return SX_STATUS_ENTRY_ALREADY_EXISTS fails because the profile key has been already created.
294  * @return SX_STATUS_RESOURCE_IN_USE fails because of the profile being bound to some ACL actions. Details see SET/DESTROY descriptions.
295  * @return SX_STATUS_ENTRY_NOT_FOUND fails because of the profile is not created. Details see SET/DESTROY descriptions.
296  */
